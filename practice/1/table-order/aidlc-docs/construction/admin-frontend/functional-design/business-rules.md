# Unit 3: Admin Frontend - 비즈니스 규칙

---

## 1. 인증 규칙

| 규칙 ID | 규칙 | 설명 |
|---|---|---|
| BR-A01 | 토큰 만료 시 자동 로그아웃 | 16시간 후 토큰 삭제 + LoginPage 리다이렉트 |
| BR-A02 | 인증되지 않은 접근 차단 | /admin/* 경로는 유효한 ADMIN 토큰 필요 |
| BR-A03 | Axios 인터셉터 토큰 주입 | 모든 API 요청에 Authorization 헤더 자동 추가 |
| BR-A04 | 401 응답 시 자동 로그아웃 | API 응답 401 수신 시 토큰 삭제 + 로그인 리다이렉트 |
| BR-A05 | 로그인 필드 필수 검증 | 매장코드, 사용자명, 비밀번호 모두 필수 |

---

## 2. 주문 상태 전이 규칙

| 규칙 ID | 규칙 | 설명 |
|---|---|---|
| BR-O01 | 유효한 상태 전이만 허용 | PENDING → PREPARING → COMPLETED (순방향만) |
| BR-O02 | 완료된 주문 상태 변경 불가 | COMPLETED 상태의 주문은 드롭다운 비활성화 |
| BR-O03 | 신규 주문 하이라이트 | NEW_ORDER 이벤트 수신 시 5초간 시각적 강조 |
| BR-O04 | 총 주문액 자동 계산 | 주문 추가/삭제 시 테이블별 총액 재계산 |

### 상태 전이 다이어그램
```
PENDING --> PREPARING --> COMPLETED
(대기중)    (준비중)      (완료)
```

## 3. 메뉴 입력 검증 규칙

| 규칙 ID | 필드 | 규칙 | 에러 메시지 |
|---|---|---|---|
| BR-M01 | 메뉴명 | 필수, 1~50자 | "메뉴명을 입력해주세요" / "50자 이내로 입력해주세요" |
| BR-M02 | 가격 | 필수, 0 이상 정수 | "가격을 입력해주세요" / "0 이상의 값을 입력해주세요" |
| BR-M03 | 카테고리 | 필수, 유효한 카테고리 ID | "카테고리를 선택해주세요" |
| BR-M04 | 설명 | 선택, 최대 200자 | "200자 이내로 입력해주세요" |
| BR-M05 | 이미지 URL | 선택, URL 형식 검증 | "올바른 URL 형식을 입력해주세요" |
| BR-M06 | 노출 순서 | 선택, 0 이상 정수 | "0 이상의 값을 입력해주세요" |
