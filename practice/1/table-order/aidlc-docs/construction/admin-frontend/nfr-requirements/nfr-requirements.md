# Unit 3: Admin Frontend - NFR 요구사항

---

## 1. 성능 요구사항

| ID | 요구사항 | 목표 |
|---|---|---|
| NFR-P01 | 초기 페이지 로드 시간 | 3초 이내 (LCP) |
| NFR-P02 | SSE 이벤트 수신 후 UI 반영 | 500ms 이내 |
| NFR-P03 | API 호출 후 UI 반영 | 1초 이내 |
| NFR-P04 | 번들 사이즈 | 초기 로드 500KB 이하 (gzip) |
| NFR-P05 | 코드 스플리팅 | 페이지별 lazy loading 적용 |

---

## 2. 사용성 요구사항

| ID | 요구사항 | 설명 |
|---|---|---|
| NFR-U01 | 반응형 레이아웃 | 태블릿(768px+) ~ 데스크톱(1280px+) 지원 |
| NFR-U02 | 에러 피드백 | 토스트 알림 (우측 상단, 3초 후 자동 사라짐) |
| NFR-U03 | 로딩 상태 | 스피너 (중앙 로딩 인디케이터) |
| NFR-U04 | 확인 다이얼로그 | 파괴적 동작(삭제, 이용 완료) 전 확인 필수 |
| NFR-U05 | 키보드 접근성 | 주요 동작 키보드로 수행 가능 |

---

## 3. 신뢰성 요구사항

| ID | 요구사항 | 설명 |
|---|---|---|
| NFR-R01 | SSE 자동 재연결 | 연결 끊김 시 3초 대기 후 재시도, 최대 5회 |
| NFR-R02 | SSE 재연결 시 데이터 동기화 | 재연결 성공 시 전체 데이터 재로드 |
| NFR-R03 | 연결 상태 표시 | SSE 연결/끊김 상태 UI 인디케이터 |
| NFR-R04 | API 에러 처리 | 모든 API 호출에 에러 핸들링 + 토스트 알림 |
| NFR-R05 | 인증 만료 처리 | 401 응답 시 자동 로그아웃 + 리다이렉트 |

---

## 4. 유지보수성 요구사항

| ID | 요구사항 | 설명 |
|---|---|---|
| NFR-M01 | TypeScript strict 모드 | 타입 안전성 보장 |
| NFR-M02 | 컴포넌트 분리 | 페이지/공통/모달 컴포넌트 명확한 분리 |
| NFR-M03 | API 클라이언트 중앙화 | Axios 인스턴스 + 인터셉터 공유 |
| NFR-M04 | 상태 관리 패턴 | Context + useReducer 일관된 패턴 |
| NFR-M05 | PBT 테스트 | reducer, 검증 로직에 property-based testing 적용 |
