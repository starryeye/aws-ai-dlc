# 테이블오더 서비스 - 요구사항 문서

## 1. 의도 분석 (Intent Analysis)

- **사용자 요청**: 테이블오더 서비스 구축 (디지털 주문 시스템)
- **요청 유형**: New Project (신규 프로젝트)
- **범위 추정**: Multiple Components (프론트엔드 + 백엔드 + DB)
- **복잡도 추정**: Moderate (다수의 기능, 실시간 통신, 세션 관리 포함)

---

## 2. 기술 스택 결정

| 영역 | 기술 | 비고 |
|---|---|---|
| 백엔드 | Java (Spring Boot) | REST API, SSE 지원 |
| 프론트엔드 | React (TypeScript) | 고객용 + 관리자용 |
| 데이터베이스 | SQLite | 경량 DB, 개발/소규모 환경 적합 |
| 배포 환경 | Docker Compose (로컬) | 개발 환경 중심 |
| 이미지 관리 | 외부 URL 참조 | 파일 업로드 없음 |
| 초기 데이터 | Seed Data | 자동 생성 |

## 3. 기능 요구사항 (Functional Requirements)

### 3.1 고객용 기능 (Customer)

#### FR-C01: 테이블 태블릿 자동 로그인 및 세션 관리
- 관리자가 1회 초기 설정 (매장 식별자, 테이블 번호, 비밀번호)
- 로그인 정보 로컬 저장 후 자동 로그인
- 16시간 세션 유지, 만료 시 관리자 수동 종료

#### FR-C02: 메뉴 조회 및 탐색
- 메뉴 화면이 기본 화면
- 카테고리별 메뉴 분류 및 표시
- 메뉴 상세 정보: 메뉴명, 가격, 설명, 이미지(외부 URL)
- 카테고리 간 빠른 이동
- 카드 형태 레이아웃, 터치 친화적 (최소 44x44px)

#### FR-C03: 장바구니 관리
- 메뉴 추가/삭제, 수량 조절
- 총 금액 실시간 계산
- 장바구니 비우기
- 클라이언트 측 로컬 저장 (새로고침 시 유지)
- 서버 전송은 주문 확정 시에만

#### FR-C04: 주문 생성
- 주문 내역 최종 확인 후 확정
- 주문 성공: 주문 번호 표시 → 장바구니 비우기 → 5초 후 메뉴 화면 리다이렉트
- 주문 실패: 에러 메시지 표시, 장바구니 유지
- 주문 정보: 매장 ID, 테이블 ID, 메뉴 목록(메뉴명, 수량, 단가), 총 금액, 세션 ID

#### FR-C05: 주문 내역 조회
- 현재 테이블 세션 주문만 표시
- 주문 시간 순 정렬
- 주문별: 주문 번호, 시각, 메뉴/수량, 금액, 상태(대기중/준비중/완료)
- 매장 이용 완료 처리된 주문 제외

### 3.2 관리자용 기능 (Admin)

#### FR-A01: 매장 인증
- 매장 식별자 + 사용자명 + 비밀번호 로그인
- JWT 토큰 기반 인증, 16시간 세션 유지
- 브라우저 새로고침 시 세션 유지
- 비밀번호 bcrypt 해싱
- 로그인 시도 제한

#### FR-A02: 실시간 주문 모니터링
- SSE 기반 실시간 주문 목록 업데이트
- 그리드/대시보드 레이아웃: 테이블별 카드 형태
- 각 테이블 카드: 총 주문액, 최신 주문 미리보기
- 주문 카드 클릭 시 전체 메뉴 목록 상세 보기
- 주문 상태 변경 (대기중/준비중/완료)
- 신규 주문 시각적 강조
- 2초 이내 주문 표시
- SSE 연결 끊김 시 자동 재연결

#### FR-A03: 테이블 관리
- 테이블 태블릿 초기 설정 (번호, 비밀번호, 16시간 세션)
- 주문 삭제 (확인 팝업 → 즉시 삭제 → 총 주문액 재계산)
- 테이블 세션 종료 (이용 완료): 주문 내역 과거 이력 이동, 주문 목록/총액 리셋
- 과거 주문 내역 조회: 시간 역순, 날짜 필터링

#### FR-A04: 메뉴 관리 (MVP 포함)
- 메뉴 CRUD (조회/등록/수정/삭제)
- 카테고리별 조회
- 메뉴 정보: 메뉴명, 가격, 설명, 카테고리, 이미지 URL
- 메뉴 노출 순서 조정
- 필수 필드 검증, 가격 범위 검증

## 4. 비기능 요구사항 (Non-Functional Requirements)

### NFR-01: 성능
- SSE 기반 실시간 통신: 주문 후 2초 이내 관리자 화면 표시
- 소규모 환경 최적화 (매장당 테이블 10개 이하)

### NFR-02: 확장성
- 소규모 (2~10개 매장) 지원 가능한 구조
- SQLite 사용으로 경량 운영

### NFR-03: 사용성
- 터치 친화적 UI (최소 44x44px 버튼)
- 직관적 카드 기반 레이아웃
- 명확한 시각적 계층 구조

### NFR-04: 데이터 관리
- 장바구니: 클라이언트 로컬 저장 (localStorage)
- 주문 이력: DB 영구 저장
- 세션 ID 기반 주문 그룹화

### NFR-05: 배포
- Docker Compose 기반 로컬 개발 환경
- 프론트엔드 + 백엔드 + DB 컨테이너 구성

## 5. 제외 범위 (Out of Scope)

요구사항 제약 문서(`requirements/constraints.md`) 기반:
- 결제 처리 (PG사 연동, 영수증, 환불, 포인트/쿠폰)
- 복잡한 인증 (OAuth, SNS, 2FA)
- 파일 업로드/리사이징
- 알림 시스템 (푸시, SMS, 이메일, 소리)
- 주방 기능 (주방 전달, 재고 관리)
- 고급 기능 (분석, 리포트, 직원 관리, 예약, 리뷰, 다국어)
- 외부 연동 (배달, POS, 소셜, 지도, 번역)
