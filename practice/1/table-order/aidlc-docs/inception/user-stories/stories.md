# 테이블오더 서비스 - 사용자 스토리

분류 방식: Feature 기반 | 세분화: 중간 단위 | 인수 기준: 체크리스트 | 우선순위: P1/P2/P3

---

## 1. 테이블 자동 로그인 및 세션 관리

### US-01: 테이블 태블릿 초기 설정
**As a** 매장 관리자  
**I want to** 태블릿에 매장 식별자, 테이블 번호, 비밀번호를 설정할 수 있다  
**So that** 고객이 별도 로그인 없이 즉시 주문할 수 있다  
**Priority**: P1  
**Persona**: 관리자 (박서연)

**Acceptance Criteria**:
- [ ] 매장 식별자 입력 필드가 존재한다
- [ ] 테이블 번호 입력 필드가 존재한다
- [ ] 테이블 비밀번호 입력 필드가 존재한다
- [ ] 설정 완료 시 로그인 정보가 로컬에 저장된다
- [ ] 설정 완료 후 메뉴 화면으로 이동한다

### US-02: 테이블 자동 로그인
**As a** 고객  
**I want to** 태블릿 접근 시 자동으로 로그인된다  
**So that** 별도 인증 절차 없이 바로 주문할 수 있다  
**Priority**: P1  
**Persona**: 고객 (김민수)

**Acceptance Criteria**:
- [ ] 저장된 로그인 정보로 자동 인증이 수행된다
- [ ] 자동 로그인 성공 시 메뉴 화면이 표시된다
- [ ] 자동 로그인 실패 시 초기 설정 화면으로 이동한다
- [ ] 세션이 16시간 유지된다

---

## 2. 메뉴 조회 및 탐색

### US-03: 카테고리별 메뉴 조회
**As a** 고객  
**I want to** 카테고리별로 분류된 메뉴를 볼 수 있다  
**So that** 원하는 메뉴를 쉽게 찾을 수 있다  
**Priority**: P1  
**Persona**: 고객 (김민수)

**Acceptance Criteria**:
- [ ] 메뉴 화면이 기본(홈) 화면으로 표시된다
- [ ] 카테고리 목록이 표시된다
- [ ] 카테고리 선택 시 해당 카테고리의 메뉴만 표시된다
- [ ] 카테고리 간 빠른 이동이 가능하다
- [ ] 카드 형태의 메뉴 레이아웃으로 표시된다

### US-04: 메뉴 상세 정보 확인
**As a** 고객  
**I want to** 메뉴의 상세 정보(이름, 가격, 설명, 이미지)를 확인할 수 있다  
**So that** 주문 전 메뉴에 대해 충분히 파악할 수 있다  
**Priority**: P1  
**Persona**: 고객 (김민수)

**Acceptance Criteria**:
- [ ] 메뉴명이 표시된다
- [ ] 가격이 표시된다
- [ ] 메뉴 설명이 표시된다
- [ ] 메뉴 이미지(외부 URL)가 표시된다
- [ ] 터치 친화적 버튼 크기 (최소 44x44px)

## 3. 장바구니 관리

### US-05: 장바구니에 메뉴 추가 및 수량 조절
**As a** 고객  
**I want to** 메뉴를 장바구니에 추가하고 수량을 조절할 수 있다  
**So that** 주문할 메뉴를 미리 선택하고 관리할 수 있다  
**Priority**: P1  
**Persona**: 고객 (김민수)

**Acceptance Criteria**:
- [ ] 메뉴 추가 버튼으로 장바구니에 메뉴를 추가할 수 있다
- [ ] 수량 증가/감소 버튼이 동작한다
- [ ] 메뉴 삭제가 가능하다
- [ ] 장바구니 비우기 기능이 있다
- [ ] 총 금액이 실시간으로 계산되어 표시된다
- [ ] 페이지 새로고침 시에도 장바구니가 유지된다 (localStorage)
- [ ] 서버 전송은 주문 확정 시에만 수행된다

---

## 4. 주문 생성

### US-06: 주문 확정 및 생성
**As a** 고객  
**I want to** 장바구니의 메뉴를 최종 확인하고 주문을 확정할 수 있다  
**So that** 실제 주문이 매장에 전달된다  
**Priority**: P1  
**Persona**: 고객 (김민수)

**Acceptance Criteria**:
- [ ] 주문 전 최종 확인 화면이 표시된다
- [ ] 주문 확정 버튼이 존재한다
- [ ] 주문 성공 시 주문 번호가 표시된다
- [ ] 주문 성공 시 장바구니가 자동으로 비워진다
- [ ] 주문 성공 후 5초 뒤 메뉴 화면으로 자동 리다이렉트된다
- [ ] 주문 실패 시 에러 메시지가 표시되고 장바구니가 유지된다
- [ ] 주문 정보에 매장 ID, 테이블 ID, 메뉴 목록, 총 금액, 세션 ID가 포함된다

---

## 5. 주문 내역 조회

### US-07: 현재 세션 주문 내역 조회
**As a** 고객  
**I want to** 현재 테이블 세션의 주문 내역을 확인할 수 있다  
**So that** 이번 방문에서 주문한 내용과 상태를 파악할 수 있다  
**Priority**: P1  
**Persona**: 고객 (김민수)

**Acceptance Criteria**:
- [ ] 현재 테이블 세션의 주문만 표시된다 (이전 세션 제외)
- [ ] 주문 시간 순으로 정렬된다
- [ ] 주문별 주문 번호, 시각, 메뉴/수량, 금액이 표시된다
- [ ] 주문 상태(대기중/준비중/완료)가 표시된다
- [ ] 매장 이용 완료 처리된 주문은 제외된다

## 6. 매장 인증

### US-08: 관리자 로그인
**As a** 매장 관리자  
**I want to** 매장 식별자, 사용자명, 비밀번호로 로그인할 수 있다  
**So that** 매장 관리 시스템에 접근할 수 있다  
**Priority**: P1  
**Persona**: 관리자 (박서연)

**Acceptance Criteria**:
- [ ] 매장 식별자, 사용자명, 비밀번호 입력 필드가 존재한다
- [ ] JWT 토큰 기반 인증이 수행된다
- [ ] 로그인 성공 시 16시간 세션이 유지된다
- [ ] 브라우저 새로고침 시 세션이 유지된다
- [ ] 16시간 후 자동 로그아웃된다
- [ ] 비밀번호는 bcrypt로 해싱되어 저장된다
- [ ] 로그인 시도 제한이 적용된다

---

## 7. 실시간 주문 모니터링

### US-09: 실시간 주문 대시보드
**As a** 매장 관리자  
**I want to** 들어오는 주문을 실시간으로 대시보드에서 확인할 수 있다  
**So that** 주문을 놓치지 않고 즉시 처리할 수 있다  
**Priority**: P1  
**Persona**: 관리자 (박서연)

**Acceptance Criteria**:
- [ ] SSE 기반으로 주문 목록이 실시간 업데이트된다
- [ ] 그리드 레이아웃으로 테이블별 카드가 표시된다
- [ ] 각 테이블 카드에 총 주문액이 표시된다
- [ ] 최신 주문 미리보기가 카드에 표시된다
- [ ] 신규 주문 시 시각적 강조(색상/애니메이션)가 적용된다
- [ ] 주문 후 2초 이내에 관리자 화면에 표시된다
- [ ] SSE 연결 끊김 시 자동 재연결된다

### US-10: 주문 상태 변경
**As a** 매장 관리자  
**I want to** 주문 상태를 대기중/준비중/완료로 변경할 수 있다  
**So that** 주문 처리 진행 상황을 관리할 수 있다  
**Priority**: P1  
**Persona**: 관리자 (박서연)

**Acceptance Criteria**:
- [ ] 주문 카드 클릭 시 전체 메뉴 목록 상세 보기가 표시된다
- [ ] 주문 상태를 대기중 → 준비중 → 완료로 변경할 수 있다
- [ ] 상태 변경이 즉시 반영된다

## 8. 테이블 관리

### US-11: 주문 삭제 (직권 수정)
**As a** 매장 관리자  
**I want to** 특정 주문을 삭제할 수 있다  
**So that** 잘못된 주문을 정정할 수 있다  
**Priority**: P1  
**Persona**: 관리자 (박서연)

**Acceptance Criteria**:
- [ ] 주문 삭제 버튼이 존재한다
- [ ] 삭제 전 확인 팝업이 표시된다
- [ ] 확인 시 주문이 즉시 삭제된다
- [ ] 테이블 총 주문액이 재계산된다
- [ ] 성공/실패 피드백이 표시된다

### US-12: 테이블 이용 완료 (세션 종료)
**As a** 매장 관리자  
**I want to** 테이블 이용 완료 처리를 할 수 있다  
**So that** 새 고객이 이전 주문 없이 깨끗한 상태로 시작할 수 있다  
**Priority**: P1  
**Persona**: 관리자 (박서연)

**Acceptance Criteria**:
- [ ] 테이블 이용 완료 버튼이 존재한다
- [ ] 완료 전 확인 팝업이 표시된다
- [ ] 해당 세션의 주문 내역이 과거 이력으로 이동된다
- [ ] 테이블 현재 주문 목록 및 총 주문액이 0으로 리셋된다
- [ ] 새 고객의 첫 주문 시 새로운 세션이 시작된다
- [ ] 성공/실패 피드백이 표시된다

### US-13: 과거 주문 내역 조회
**As a** 매장 관리자  
**I want to** 테이블별 과거 주문 내역을 조회할 수 있다  
**So that** 이전 고객의 주문 이력을 확인할 수 있다  
**Priority**: P2  
**Persona**: 관리자 (박서연)

**Acceptance Criteria**:
- [ ] 과거 내역 버튼이 존재한다
- [ ] 테이블별 과거 주문 목록이 시간 역순으로 표시된다
- [ ] 각 주문에 주문 번호, 시각, 메뉴 목록, 총 금액, 이용 완료 시각이 표시된다
- [ ] 날짜 필터링 기능이 있다
- [ ] 닫기 버튼으로 대시보드로 복귀할 수 있다
